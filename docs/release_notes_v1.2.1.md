# Release Notes - v1.2.1

## 版本信息

- 版本号：`v1.2.1`
- 发布时间：`2026-06-15`
- 对应提交：`39bbbc9 Finalize report layout and action grid`

## 本次版本概览

`v1.2.1` 是本项目从联调状态收口到正式版的一次发布，重点不在新增数据源，而在于把已有链路稳定下来，并把月报展示、钉钉发送和版本输出整理成可长期复用的正式流程。

本次版本主要覆盖三类内容：

1. 月报命名和输出口径正式固定。
2. 钉钉 PDF 群文件发送链路收口为可持续复用的正式路径。
3. 报告版式重构为更适合管理阅读的正式版样式。

## 主要更新

### 1. 报告输出正式化

- 月报文件名固定为 `YYYYMM-月度纸袋分析报告`。
- 报告首页标题同步固定为 `YYYYMM-月度纸袋分析报告`。
- 不再使用运行时随机后缀作为正式产出文件名。

影响：

- 便于归档、转发、复盘和群文件检索。
- 便于后续钉钉机器人、群助手和人工分析使用统一文件名。

### 2. 钉钉发送链路正式化

- 正式路径统一为：企业内部应用获取权限后，通过 `convFile` 将 PDF 作为群文件发送。
- 新增并收口以下绑定要素的采集与存储：
  - `openConversationId`
  - `chatId`
  - `unionId`
  - `userId`
  - `operatorName`
- 保留本地 H5 JSAPI 绑定页和签名服务，作为一次性绑定工具。
- 已生成报告支持通过 `send-existing-report` 直接复用绑定重发。

影响：

- 后续每月定时任务不需要再次人工选择群。
- 正式群文件链路与“文件分析”场景更一致，减少二次上传的重复操作。

### 3. 报告版式正式化

#### 核心结论与摘要重构

- “经营摘要”保留为首页快速导读。
- “核心结论”改为管理判断视图，不再简单重复摘要内容。
- 拆分为：
  - `库存与趋势判断`
  - `大区与结构风险`
  - `订购与终端执行`

#### 本月优先执行事项重构

- 顶部保留三类摘要卡：
  - `库销与库存类`
  - `纸袋使用合规类`
  - `终端异常 / 盘点类`
- 下方动作卡统一为 3 列等高网格。
- 每张动作卡固定显示：
  - 类别标签
  - `P1 / P2` 优先级
  - 执行区域
  - 问题类型
  - 首要动作
  - 复盘指标

#### AI 重点行动清单重构

- 按 `P1` 和 `P2` 分组展示。
- 每组内部再按动作类别拆分。
- 增加动作总览卡，便于先看总量、再看明细。

#### 文案高亮优化

- 报告中冒号前字段名、关键数值、重点关键词已统一加粗。
- 重点关键词包括但不限于：
  - `P1 / P2`
  - `红灯 / 黄灯 / 绿灯`
  - `高风险`
  - `异常订单`
  - `库存积压`
  - `补货压力`

## 修复项

- 修复报告标题和文件名不一致的问题。
- 修复正式文件名中携带随机后缀的问题。
- 修复“核心结论”与“经营摘要”内容重复的问题。
- 修复“地区焦点”和“专项焦点”排版不一致的问题。
- 修复“本月优先执行事项”卡片长短不齐的问题。
- 修复重复问题标签堆叠的问题，例如：
  - `库存结构不科学 / 库存结构不科学`
  - 黄灯提示中同类库存问题连续重复

## 使用说明

### 正式版推荐命令

生成指定月份报告但不发送：

```bash
python3 -m src.main run-once --month 2026-05 --skip-send
```

直接发送已生成 PDF：

```bash
python3 -m src.main send-existing-report --month 2026-05
```

检查钉钉正式发送链路：

```bash
python3 -m src.main check-dingtalk --month 2026-05
```

### 升级建议

- 如果你已经完成过群绑定，本版本会继续复用已有 `data/processed/dingtalk_binding.json`。
- 如果尚未完成群绑定，先执行 H5 绑定流程，保存 `openConversationId` 等信息后，再进行正式发送。
- 如果此前依赖 webhook 文本发送作为主链路，建议改为当前正式版的 PDF 群文件发送链路。
